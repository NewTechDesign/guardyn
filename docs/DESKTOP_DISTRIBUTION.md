# Guardyn Desktop Distribution Guide

This guide covers building, signing, and distributing the Guardyn desktop client.

## Overview

Guardyn Desktop is built with [Tauri](https://tauri.app/) - a secure framework for building native applications with web technologies. It provides:

- **Small bundle size** - ~10MB compared to 100MB+ for Electron
- **Native performance** - Rust backend, WebView frontend
- **Security-first** - Sandboxed by default, no Node.js runtime
- **Auto-updates** - Built-in updater mechanism

## Build Requirements

### Linux Build Environment

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install -y \
    build-essential \
    curl \
    wget \
    libwebkit2gtk-4.1-dev \
    libssl-dev \
    libayatana-appindicator3-dev \
    librsvg2-dev \
    dpkg-dev \
    file

# Install Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Install Node.js 18+
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# Install Tauri CLI
cargo install tauri-cli
```

### Recommended: Use Nix Shell

```bash
# From project root
nix develop

# All tools are pre-configured
```

## Building

### Quick Build (Linux)

```bash
cd client-desktop

# Development build
cargo tauri dev

# Release build
./scripts/build-linux.sh linux release
```

### Full Release Build

```bash
cd client-desktop

# Build with signing
SIGN=true ./scripts/build-release.sh
```

### Build Options

| Option | Default | Description |
|--------|---------|-------------|
| `CLEAN=true` | false | Clean build (removes previous artifacts) |
| `SIGN=true` | false | GPG sign packages |
| `AUTO_INSTALL=true` | false | Auto-install missing dependencies |

## Signing

### GPG Signing

For release builds, sign packages with GPG:

```bash
# Generate key (if needed)
gpg --gen-key

# Sign release
GPG_KEY_ID="your-key-id" SIGN=true ./scripts/build-release.sh
```

### Tauri Code Signing

For auto-updater signature verification:

```bash
# Generate Tauri keypair
cargo tauri signer generate -w ~/.tauri/guardyn.key

# Set environment for builds
export TAURI_SIGNING_PRIVATE_KEY_PATH=~/.tauri/guardyn.key
export TAURI_SIGNING_PRIVATE_KEY_PASSWORD="your-password"
```

## Auto-Updater

Guardyn Desktop supports automatic updates via Tauri's built-in updater.

### Configuration

Update `src-tauri/tauri.conf.json`:

```json
{
  "plugins": {
    "updater": {
      "active": true,
      "endpoints": [
        "https://releases.guardyn.io/desktop/latest.json"
      ],
      "dialog": true,
      "pubkey": "YOUR_PUBLIC_KEY"
    }
  }
}
```

### Update Manifest

The `latest.json` manifest is generated by `build-release.sh`:

```json
{
  "version": "1.0.0",
  "notes": "Guardyn Desktop v1.0.0",
  "pub_date": "2024-01-01T00:00:00Z",
  "platforms": {
    "linux-x86_64": {
      "signature": "BASE64_SIGNATURE",
      "url": "https://github.com/AoT-Technologies/guardyn/releases/download/v1.0.0/guardyn-1.0.0.AppImage"
    }
  }
}
```

## Verification

Users should verify downloads:

```bash
# Download checksums and signature
wget https://github.com/.../SHA256SUMS.txt
wget https://github.com/.../SHA256SUMS.txt.asc

# Verify GPG signature
gpg --verify SHA256SUMS.txt.asc SHA256SUMS.txt

# Verify checksum
sha256sum -c SHA256SUMS.txt
```

## CI/CD Integration

### GitHub Actions

Release workflow in `.github/workflows/desktop-release.yml`:

```yaml
name: Desktop Release

on:
  push:
    tags:
      - 'desktop-v*'

jobs:
  build-linux:
    runs-on: ubuntu-22.04
    steps:
      - uses: actions/checkout@v4
      
      - name: Install dependencies
        run: |
          sudo apt update
          sudo apt install -y libwebkit2gtk-4.1-dev
          
      - name: Build
        run: |
          cd client-desktop
          ./scripts/build-linux.sh linux release
          
      - name: Upload artifacts
        uses: actions/upload-artifact@v4
        with:
          name: linux-builds
          path: dist/desktop/linux/*
```

## Troubleshooting

### Common Issues

**WebKitGTK not found:**
```bash
sudo apt install libwebkit2gtk-4.1-dev
```

**AppImage won't run:**
```bash
chmod +x guardyn-*.AppImage
# Or extract and run
./guardyn-*.AppImage --appimage-extract
./squashfs-root/AppRun
```

**FUSE not available (for AppImage):**
```bash
sudo apt install fuse libfuse2
```

**Build fails on Wayland:**
```bash
# Ensure X11 libraries are available
sudo apt install libxcb-shape0-dev libxcb-xfixes0-dev
```

### Debug Build

For troubleshooting issues:

```bash
# Build with debug symbols
cargo tauri build --debug

# Enable verbose logging
RUST_LOG=debug ./guardyn
```

## Security Considerations

1. **Always verify checksums** before installing
2. **Only download from official sources** (GitHub Releases)
3. **Check GPG signatures** on release assets
4. **Auto-updater verifies signatures** before applying updates
5. **Report vulnerabilities** to security@guardyn.io
